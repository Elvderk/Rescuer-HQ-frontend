**Статус:** Production-Ready Specification  
**Слой:** `features/tasks`  
**Технологический стек:** Riverpod Providers, Drift SQLite, WebSocket Event Dispatcher, Dio API client

## 1. Feature Architecture Overview

Управление задачами (объекты `TaskModel`) в спасательной операции охватывает распределение ресурсов (личный состав, беспилотники, патрульные машины). Архитектура строго разделена по классическим Clean слоям:

```text
 ┌────────────────────────────────────────────────────────────────────────┐
 │                           PRESENTATION LAYER                           │
 │  - TasksListScreen (Фильтры по приоритетам: LOW, NORMAL, HIGH, CRITICAL) │
 │  - TaskDetailsScreen (Интерактивная лента отчетов, вложения и фото)      │
 │  - CreateTaskScreen (Мастер создания локальных и глобальных задач)       │
 └───────────────────────────────────┬────────────────────────────────────┘
                                     ▼
 ┌────────────────────────────────────────────────────────────────────────┐
 │                           APPLICATION LAYER                            │
 │  - TasksListNotifier (Реактивное вычисление списков и пагинация)       │
 │  - TaskDetailsNotifier (Активность, комментарии и статус сдачи отчетов) │
 └───────────────────────────────────┬────────────────────────────────────┘
                                     ▼
 ┌────────────────────────────────────────────────────────────────────────┐
 │                              DOMAIN LAYER                              │
 │  - TaskModel (Сущность задачи: заголовок, приоритет, ответственный)      │
 │  - TaskStatus (Enum: OPEN, IN_PROGRESS, BLOCKED, COMPLETED, CANCELLED)  │
 │  - TaskPriority (Enum: LOW, NORMAL, HIGH, CRITICAL)                    │
 └───────────────────────────────────┬────────────────────────────────────┘
                                     ▼
 ┌────────────────────────────────────────────────────────────────────────┐
 │                               DATA LAYER                               │
 │  - TasksRepository (Слияние Drift SQLite и внешних REST эндпоинтов)     │
 │  - TasksCacheDataSource (Быстрое локальное индексирование задач)        │
 └────────────────────────────────────────────────────────────────────────┘
```

## 2. Global vs Local Search Tasks Boundary

Система разграничивает задачи по области охвата:
* **Global Tasks**: Не связаны с конкретной пропажей (например, "Обслуживание генераторов штаба", "Дежурство на радиовышке"). Видны всем зарегистрированным волонтерам отряда в соответствующей вкладке.
* **Local Search Tasks**: Жёстко привязаны к операции `searchId` (например, "Обход заброшенного элеватора в квадрате В-3", "Расклейка ориентировок у ЖД Станции"). Доступны для просмотра только участникам текущего поиска.

---

## 3. Strict RBAC Policies for Lifecycle Events

Изменение жизненного цикла задачи регулируется политикой ограничений ролей (Role-Based Access Control):

* **Создание (Create) / Назначение исполнителя (Assign)**: Разрешено только ролям `Coordinator`, `Senior Coordinator`, `Director`. Обычный волонтер не имеет возможности создавать задачи для других.
* **Взятие в работу (Acquire/Take)**: Волонтер может самостоятельно взять свободную задачу (`status == Open`) со статусом `IN_PROGRESS` и назначить себя исполнителем.
* **Завершение (Complete/Close)**: Волонтер рапортует о выполнении, прикладывая текстовый отчет и фото ориентировки. Переход в статус `COMPLETED` валидируется координатором штаба.

---

## 4. Real-time Subscription Loop & Local Outbox Replays

Синхронизация задач работает бесшовно благодаря союзу локальной базы и сокетов:

1. **Inbound Transmissions**:
   Диспетчер сокета `WebSocketEventDispatcher` принимает фреймы событий:
   - `task.created` -> Добавление задачи во встроенную Drift базу.
   - `task.status.changed` / `task.assigned` -> Обновление атрибутов; UI реагирует вызовом `ref.invalidate(tasksListProvider)`.
2. **Outbound Queue Replays**:
   При оффлайн-смене статуса "В работе" -> "Выполнено", задача переходит в промежуточное локальное состояние `SyncStatus.pending`, а в фоновую локальную очередь `SyncQueueTable` помещается событие оптимистичной репликации с уникальным UUID ключом идемпотентности. При обнаружении интернета `SyncEngine` проливает накопленные мутации на сервер, исключая потери отчетов.

## 5. Media Attachments & Report Evidence Support

Выполнение задач по расклейке ориентировок требует строгой отчетности:
- К задаче прикрепляются локальные ссылки на сделанные фотографии.
- Фотографии пережимаются встроенным плагином сжатия во избежание перегрузки радиоканала.
- Локальный массив путей к файлам `pending_photos_queue` передается фоновой службе доставки MinIO/S3, гарантируя догрузку улик даже после перезапуска приложения волонтера.
