# Rescuer HQ — Districts, Maps & Live Geolocation Architecture
**Статус:** Production-Ready Specification  
**Слой:** `features/districts` & `features/geo`  
**Технологический стек:** Google Maps Flutter, Geolocator, ConnectivityPlus, WebSocket, Riverpod Provider

## 1. GIS & Coordinate Management Layer (Maps Provider)

Управление гео-координатами в мобильном приложении Rescuer HQ построено по модульному принципу:
- **Provider Agnostic Geometry**: Для отрисовки карт создана абстракция `MapControllerFacade` и `MapGeometryAdapter`. Это позволяет легко заменять базовый провайдер карт (`Google Maps SDK` на `Yandex MapKit` или `OSM flutter_map`) без переписывания бизнес-логики.
- **Unified Projections**: Все полигоны районов, треки волонтеров и маркеры происшествий транслируются из формата GeoJSON / WKT в независимые иммутабельные сущности `AppPolygon` и `AppMarker`.

```text
 ┌────────────────────────────────────────────────────────────────────────┐
 │                           PRESENTATION LAYER                           │
 │  - MapScreen (Главный экран поисковой карты со слоями треков и SOS)    │
 │  - DistrictsScreen (Иерархический список и статусы секторов поиска)    │
 │  - DistrictDetailsScreen (Детальная карточка района с границами)       │
 └───────────────────────────────────┬────────────────────────────────────┘
                                     ▼
 ┌────────────────────────────────────────────────────────────────────────┐
 │                           APPLICATION LAYER                            │
 │  - TrackingNotifier (Фоновый геотрекинг и отправка координат)          │
 │  - DistrictNotifier (Бизнес-логика резервирования и сдачи районов)     │
 └───────────────────────────────────┬────────────────────────────────────┘
                                     ▼
 ┌────────────────────────────────────────────────────────────────────────┐
 │                              DOMAIN LAYER                              │
 │  - DistrictModel (Районы с полигонами координат, площадями и статусом) │
 │  - LiveLocation (Строго типизированная геоточка с точностью GPS)       │
 │  - AppPolygon / AppMarker (Агностичные геометрические примитивы)       │
 └───────────────────────────────────┬────────────────────────────────────┘
                                     ▼
 ┌────────────────────────────────────────────────────────────────────────┐
 │                               DATA LAYER                               │
 │  - GeolocationService (Интеграция Geolocator, фоновые разрешения)      │
 │  - DistrictsRepository (Интеграция API, Drift SQLite кэширование)      │
 └────────────────────────────────────────────────────────────────────────┘
```

## 2. District State Machine & Ownership Operations

Районы делятся по координационной сетке поискового штаба (например, Лесной сектор А-1, Квадрат Б-4).
Каждый район проходит строгий жизненный цикл:

```text
               ┌───────────────┐
               │     FREE      │
               └───────┬───────┘
                       │ Action: Take District (Волонтер выразил согласие)
                       ▼
               ┌───────────────┐
               │  IN_PROGRESS  │ ◄═══► Live Geolocation tracking of squad
               └───────┬───────┘
                       │ Action: Submit Report / Complete (Район обследован)
                       ▼
               ┌───────────────┐
               │   COMPLETED   │
               └───────────────┘
                       ▲
                       │ Action: Reset (Координатор решает прочесать заново)
                       ▼
               ┌───────────────┐
               │    BLOCKED    │ (Например, обнаружена зона диких животных / обрыв)
               └───────────────┘
```

## 3. Realtime Location Tracking & Battery Recovery Strategy

Отслеживание координат волонтеров — критический элемент безопасности при лесных спасательных операциях.

### 1. Настройки Геолокации (Location Settings Contract)
- **Частота**: Каждые **15 метров** или **10 секунд** во время движения.
- **Точность**: `LocationAccuracy.high` в активной зоне; `LocationAccuracy.medium` при нахождении в штабе (определяется геозоной Geofence).
- **Background Mode**: Постоянная фоновая служба с нестираемым системным уведомлением (Foreground Service), предотвращающим уничтожение процесса операционной системой Android (Low Memory Killer).

### 2. Алгоритм энергосбережения в поле (Battery Optimization Rule)
- Если волонтер не движется (простой регистрируется встроенным акселерометром смартфона), частота отправки GPS-пакетов снижается до 5 минут.
- Если заряд аккумулятора падает ниже **20%**, точность геолокатора автоматически деградирует до `LocationAccuracy.balanced`, выключая ресурсоемкий чип GPS и переключаясь на триангуляцию по вышкам LBS/Cell-ID для сохранения автономности устройства.

## 4. SOS Pulse & Emergency Event Priority

При нажатии волонтером тревожной кнопки SOS во всем поисковом отряде:
1. **Pulse Marker Overlay**: На интерактивной карте всех участников и координатора загорается ярко-красный SOS маркер, пульсирующий с частотой 1.5 Гц (раз в 650мс).
2. **Forced Socket Handshake**: WebSocket прекращает буферизацию геотреков и переходит на повышенный приоритет трансляции сотовых фреймов.
3. **Emergency Routing**: Встроенная система вычисления маршрутов (`RouteBuilder`) рассчитывает кратчайший пеший или автомобильный маршрут от местоположения спасателя до точки SOS бедствия с использованием оффлайн-карт OsmAnd / Yandex / Mapbox.

## 5. Offline GIS Cache Strategy & Map Providers Failover

В глухом лесу, где сотовая связь отсутствует, карты продолжают работать по следующему алгоритму:
- **Districts Layer Caching**: Координаты полиметров секторов берутся напрямую из реляционных таблиц Drift SQLite.
- **Geocoding Reverse Cache**: Все координаты преобразуются в текстовые адреса локально: если в кэше есть объект, обращение к внешнему серверу номинации (Nominatim, Yandex) не производится.
- **External Maps Redirection**: При нажатии «Открыть в навигаторе», приложение формирует универсальные URI схемы для мобильного устройства:
  - Yandex Maps: `yandexmaps://maps.yandex.ru/?rtext=...`
  - 2GIS: `dgis://2gis.ru/routeSearch/...`
  - Google Maps: `https://www.google.com/maps/dir/?api=1&...`
Это обеспечивает мгновенную загрузку нативного навигатора волонтера со скачанными локально оффлайн-картами.
